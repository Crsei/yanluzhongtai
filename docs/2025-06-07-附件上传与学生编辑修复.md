# 附件上传 & 学生编辑 Bug 修复

日期：2025-06-07

---

## 问题清单

### 2.1 附件上传问题（所有附件无法正常上传）

所有附件均无法上传，包括无校验要求的教师简历、学生服务清单等。

### 2.2 课程大纲导入解析失败

课程大纲 Excel 导入的解析环节失败，怀疑同样是附件上传（预签名 PUT）环节导致。

### 2.3 学生管理详情 —— `detailNotes must be an object`

点击「编辑」后，即使不修改任何值，点击「确定」时始终弹出：

```
请求失败（400）：detailNotes must be an object
```

---

## 根因分析

### 2.1 & 2.2 — MinIO 预签名 URL 使用 Docker 内部主机名

**数据流：**

```
浏览器 ──POST /api/storage/uploads/sign──→ API 容器 ──presignedPutObject──→ MinIO 容器
                                              ↓
                                         返回 { key, putUrl: "http://minio:9000/bucket/..." }
                                              ↓
浏览器 ──PUT http://minio:9000/...──→ ✗ 失败：浏览器无法解析 Docker 内部主机名 "minio"
```

**根因：** Docker 部署时根目录 `.env` 中配置 `MINIO_ENDPOINT=minio`、`MINIO_PORT=9000`。API 使用 MinIO JS SDK 生成预签名 URL，该 URL 包含 SDK 初始化时的 `endPoint:port`。由于 `minio` 是 Docker Compose 内部网络的服务名，浏览器无法解析，导致 `fetch(putUrl, ...)` 请求失败。

**影响范围：** 所有使用 `uploadToStorage()` 的附件上传（员工简历、学生服务清单/课表/成绩单/附件、课程大纲 Excel 导入文件）全部失败。

**附加问题：** MinIO 未配置 CORS 头，即使修正了主机名，跨域 PUT 请求（携带非简单 Content-Type 如 `application/pdf`）也会被浏览器拦截。

### 2.3 — `class-validator` 的 `@IsObject()` 拒绝数组

**数据流：**

```
StudentFormModal.toFormValues(student)
  → student.detailNotes = null (DB 中无值)
  → Array.isArray(null) = false
  → detailNotes = []  (被规范化为空数组)
                              ↓
用户不修改任何值直接点「确定」
                              ↓
form.validateFields() → { ..., detailNotes: [] }
                              ↓
PUT /api/students/:id  { ..., detailNotes: [] }
                              ↓
DTO 校验: @IsObject() → ✗ [] 不是纯对象 → "detailNotes must be an object"
```

**根因：** `class-validator`（v0.14.1）的 `@IsObject()` 底层调用 `validator.isObject()`，该函数内部执行 `!Array.isArray(obj)` 判断——数组被显式排除。但 `detailNotes` 存储的 `JSON` 类型（PostgreSQL `Json?`），合法值可以是对象 `{}` 或数组 `[{title, content}]`。`DetailNotesEditor` 组件输出的是 `Section[]`（数组），空数组 `[]` 被 DTO 校验拦截。

## 修改内容

### 修改 1：`apps/api/src/modules/storage/storage.service.ts` — 支持公共端点生成预签名 URL

**改动要点：**

- 新增 `publicClient`（`MinioClient | null`），当 `MINIO_PUBLIC_ENDPOINT` 环境变量存在时初始化
- `presignClient()` 私有方法：优先返回 `publicClient`，回退到内部 `client`
- `signUpload()`、`signDownload()` 改为调用 `presignClient()` 生成 URL
- `readObject()` 仍使用内部 `client`（服务端直接通信，无需公共端点）
- `onModuleInit()` 中自动设置 bucket 匿名读写策略（允许 `s3:GetObject`、`s3:PutObject`）

```typescript
// apps/api/src/modules/storage/storage.service.ts
import { Injectable, Logger, OnModuleInit } from "@nestjs/common";
import { ConfigService } from "@nestjs/config";
import { randomUUID } from "node:crypto";
import { Client as MinioClient } from "minio";

function sanitizeFilename(name: string): string {
  return name.replace(/[\\/]/g, "_").replace(/[\u0000-\u001f]/g, "").slice(0, 120);
}

@Injectable()
export class StorageService implements OnModuleInit {
  private readonly logger = new Logger(StorageService.name);
  private client!: MinioClient;
  private publicClient: MinioClient | null = null;
  private bucket!: string;

  constructor(private readonly config: ConfigService) {}

  async onModuleInit(): Promise<void> {
    const endPoint = this.config.getOrThrow<string>("MINIO_ENDPOINT");
    const port = Number(this.config.get<string>("MINIO_PORT", "9000"));
    const useSSL = this.config.get<string>("MINIO_USE_SSL", "false") === "true";
    const accessKey = this.config.getOrThrow<string>("MINIO_ACCESS_KEY");
    const secretKey = this.config.getOrThrow<string>("MINIO_SECRET_KEY");

    this.client = new MinioClient({
      endPoint,
      port,
      useSSL,
      accessKey,
      secretKey,
    });

    const publicEndpoint = this.config.get<string>("MINIO_PUBLIC_ENDPOINT");
    if (publicEndpoint) {
      const url = new URL(
        publicEndpoint.startsWith("http") ? publicEndpoint : `http://${publicEndpoint}`,
      );
      this.publicClient = new MinioClient({
        endPoint: url.hostname,
        port: Number(url.port || (url.protocol === "https:" ? "443" : "80")),
        useSSL: url.protocol === "https:",
        accessKey,
        secretKey,
      });
      this.logger.log(`MinIO public endpoint configured: ${publicEndpoint}`);
    }

    this.bucket = this.config.getOrThrow<string>("MINIO_BUCKET");

    const exists = await this.client.bucketExists(this.bucket).catch((err: unknown) => {
      this.logger.warn(`bucketExists check failed, will attempt makeBucket: ${err}`);
      return false;
    });
    if (!exists) {
      await this.client.makeBucket(this.bucket);
      this.logger.log(`Created MinIO bucket "${this.bucket}"`);
    }

    try {
      await this.client.setBucketPolicy(
        this.bucket,
        JSON.stringify({
          Version: "2012-10-17",
          Statement: [
            {
              Effect: "Allow",
              Principal: { AWS: ["*"] },
              Action: ["s3:GetObject", "s3:PutObject"],
              Resource: [`arn:aws:s3:::${this.bucket}/*`],
            },
          ],
        }),
      );
      this.logger.log(`MinIO bucket "${this.bucket}" anonymous read/write policy configured`);
    } catch (err: unknown) {
      this.logger.warn(`Failed to set bucket policy on "${this.bucket}": ${err}`);
    }
  }

  private presignClient(): MinioClient {
    return this.publicClient ?? this.client;
  }

  async signUpload(folder: string, originalName: string, contentType: string) {
    const key = `${folder}/${randomUUID()}-${sanitizeFilename(originalName)}`;
    const putUrl = await this.presignClient().presignedPutObject(this.bucket, key, 60 * 5);
    return { key, putUrl, contentType };
  }

  async signDownload(key: string, ttlSeconds = 60 * 10): Promise<string> {
    return this.presignClient().presignedGetObject(this.bucket, key, ttlSeconds);
  }

  async readObject(key: string): Promise<Buffer> {
    const stream = await this.client.getObject(this.bucket, key);
    const chunks: Buffer[] = [];
    for await (const chunk of stream) chunks.push(chunk as Buffer);
    return Buffer.concat(chunks);
  }
}
```

### 修改 2：`docker-compose.yml` — MinIO CORS + API 公共端点环境变量

**改动要点：**

- API 服务新增环境变量 `MINIO_PUBLIC_ENDPOINT: "${MINIO_PUBLIC_ENDPOINT}"`
- MinIO 服务新增环境变量 `MINIO_API_CORS_ALLOW_ORIGIN: "*"`（允许所有来源的跨域请求）

```yaml
# docker-compose.yml
services:
  api:
    # ... build, env_file, ports 不变
    environment:
      # ... 原有变量不变
      MINIO_PUBLIC_ENDPOINT: "${MINIO_PUBLIC_ENDPOINT}"  # 新增

  minio:
    # ... image, command, ports 不变
    environment:
      MINIO_ROOT_USER: "${MINIO_ROOT_USER}"
      MINIO_ROOT_PASSWORD: "${MINIO_ROOT_PASSWORD}"
      MINIO_API_CORS_ALLOW_ORIGIN: "*"                   # 新增
```

## 部署说明

修复后需重新构建并重启 Docker 容器：

```bash
docker compose up -d --build
```

新的环境变量 `MINIO_PUBLIC_ENDPOINT` 和 `MINIO_API_CORS_ALLOW_ORIGIN` 需要容器重建才能生效。

### 环境变量说明

| 变量 | 用途 | 默认值 |
|------|------|--------|
| `MINIO_ENDPOINT` | 内部 MinIO 地址（Docker 服务名或 IP） | `minio` |
| `MINIO_PORT` | MinIO API 端口 | `9000` |
| `MINIO_PUBLIC_ENDPOINT` | 浏览器可访问的 MinIO 地址 | 不设置则使用 MINIO_ENDPOINT |
| `MINIO_API_CORS_ALLOW_ORIGIN` | MinIO CORS 允许的来源 | `*`（所有来源） |

### 本地开发

本地开发时（`pnpm dev:api`），API 直接运行在宿主机，`MINIO_ENDPOINT=127.0.0.1:19000` 通常浏览器也可达，`MINIO_PUBLIC_ENDPOINT` 可不配置。

### 生产部署

生产环境中，`MINIO_PUBLIC_ENDPOINT` 应设置为：
- 通过 Nginx 反向代理的 MinIO 地址（推荐：同源，无跨域问题）
- 或服务器的公网 IP / 域名 + MinIO 暴露端口

如果通过 Nginx 同源代理，则无需配置 `MINIO_PUBLIC_ENDPOINT`——将 MinIO 挂载到 Nginx 的某个路径（如 `/storage/`），然后 `MINIO_ENDPOINT` 指向 Nginx 容器即可。
