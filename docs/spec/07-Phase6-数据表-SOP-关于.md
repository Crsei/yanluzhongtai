# 07. Phase 6 - 数据表 / SOP / 关于

## 1. 阶段目标

为内部团队提供统一的快捷入口中心、制度文档中心和系统说明页，同时把日志访问纳入权限控制。

## 2. 本阶段范围

- 数据表页
- SOP 页
- 关于页
- 日志入口

## 4. 数据表页规格

依据图20、图21：

- 页面标题：`数据表快捷跳转中心`
- 按业务分组展示，如“企业内部数据表”“高途合作数据表”
- 每组使用三列卡片网格
- 每张卡片是白底圆角长方块
- 顶部右侧有四个按钮：排序、添加、编辑、删除
- hover 时卡片出现蓝色细边框，同时轻微上浮和加深阴影

## 5. SOP 页规格

依据图22、图23：

- 页面标题：`标准作业程序`
- 也采用三列卡片网格
- 卡片默认视觉与数据表页一致
- hover 时边框高亮改为绿色
- 页面顶部也需要同样的四个管理按钮

## 6. 关于页规格

依据图24：

- 关于页沿用主框架布局
- 中部展示 Logo、平台名称、版本号、所属企业、问题反馈邮箱
- 页面中部提供“查看中台日志”链接入口
- 该入口仅管理员和超级管理员可访问
- 页面底部预留备案号和版权信息位置
- 正式版版本号从 `1.0.0` 起算

## 7. 预置数据要求

### 7.1 数据表

至少应初始化以下业务组：

- 企业内部数据表
- 高途合作数据表

### 7.2 SOP

至少应初始化以下入口：

- 服务全流程协同SOP
- 人员招募SOP
- 客户转化SOP
- 学管、规划师、授课老师工作流程图
- 学管&规划师如何配合
- 学管如何开展工作

### 7.3 复制类链接

部分条目不是直接跳转，而是“点击复制”链接。此类项需要：

- 复制指定 URL 到剪贴板
- 返回成功/失败提示

## 8. 权限规则

- 数据表页：登录用户可访问
- SOP：访客也可访问
- 关于：访客可访问
- 中台日志入口：仅管理员、超级管理员可访问

## 9. 本阶段验收标准

- 数据表与 SOP 都以卡片网格形式展示
- 数据表 hover 为蓝边，SOP hover 为绿边
- 每个分组都具备排序、添加、编辑、删除能力
- 关于页能展示版本、公司、邮箱和日志入口
- 访客不可直接查看日志
