# 05. Phase 4 - 课程信息与学生选课

## 1. 阶段目标

把课程、老师、学生三方关系打通，形成中台最核心的业务主链路。

## 2. 本阶段范围

- 课程列表
- 查看课程
- 添加课程
- 编辑课程
- 删除课程
- Excel 导入课程
- 选课学生选择器
- 课程状态自动计算
- 高级搜索页

## 4. 列表页规格

依据图13：

- 页面标题：`学生选课与课程信息管理`
- 左侧工具按钮为：查看课程信息、编辑课程信息、添加课程、删除课程
- `从 Excel 导入` 按钮插在删除按钮与搜索框之间
- 搜索框位于中部偏右
- 最右侧为“高级搜索”
- 页面右上角单独放置“课程大纲”跳转按钮
- 列表字段至少包括：课程编号、课程名称、所属板块、计划授课时间、课程状态、授课方式、实际授课老师
- 列表按计划授课时间降序排列

## 5. 添加/编辑课程弹窗规格

依据图14：

- 弹窗首先出现“来自课程大纲版本”的选择，默认取最新版本
- 表单采用双列布局
- 课程编号只读，保存后自动生成
- 课程所属板块和二级课程类别联动
- “选课学生”不是普通输入框，而是“只读文本框 + 右侧选择按钮”
- 该只读文本框宽度应接近整行，便于展示多个学生
- “建议授课方式”为只读字段，从大纲自动带出
- “实际授课老师”通过员工选择器指定
- “课程状态”为自动测算只读字段
- “授课课时”为自动计算只读字段
- 表单较长，内部允许滚动

## 6. 选择学生弹窗规格

依据图15：

- 弹窗标题：`选择学生`
- 顶部有搜索输入框
- 中部列表展示候选学生，按复选框选择
- 右下角按钮为“取消 / 确认选择”

## 7. 高级搜索页规格

依据图16：

- 不用普通后台表格风格，而采用较强设计感的筛选面板
- 页面可使用浅蓝渐层、毛玻璃或半透明大卡片
- 筛选项按两列网格布局
- 单项输入控件高度一致
- 底部设置通栏主查询按钮
- 支持动态增加查询条件

高级搜索字段至少包含：

- 课程名称
- 所属清单内课程名称
- 课程所属板块
- 实际授课形式
- 上课学生
- 实际授课老师

## 8. 数据与联动规则

- 课程编号格式：`TTKKYYNNN`
- `TT` 为课程板块字母标识
- `KK` 为该板块下二级课程类别序号
- `YY` 为计划授课时间年份后两位
- `NNN` 为在同一 `TT+KK+YY` 组合下的独立递增序号
- 删除后不回收流水号
- 每条课程记录表示一节具体课程
- 课程记录需维护：
  - 课程详情本身
  - 课程关联的学生列表
  - 学生关联的课程列表
  - 员工关联的课程列表

## 9. 课程状态规则

| 条件 | 状态 |
| --- | --- |
| 未填写计划授课时间 | 未排期 |
| 已填计划授课时间，未填授课时长，当前日期未到计划日期 | 已排期 |
| 已填计划授课时间，未填授课时长，当前日期等于或晚于计划日期 | 进行中 |
| 已填计划授课时间且已填授课时长 | 已完成 |

## 10. 课时规则

- 授课时长单位为 `min`
- `1课时 = 45min`
- 授课课时自动计算，四舍五入保留两位小数

## 11. 本阶段验收标准

- 可新增、编辑、删除课程
- 课程编号生成正确且删除后不回收
- 添加课程时默认绑定最新大纲版本
- 板块和二级课程类别联动正确
- 选课学生和授课老师都能通过对话框选择
- 课程状态和课时能自动计算
- 高级搜索页具备独立视觉风格和多条件筛选能力
