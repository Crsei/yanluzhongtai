# 00. 全局约束与实施路线

## 1. 项目目标

研录教学管理中台是一个仅面向企业内部使用的 Web 管理平台，目标是把员工、学生、课程、薪酬、数据表、SOP、日志统一纳入一个受权限控制的系统中，并同时兼顾 PC 和移动端浏览器可用性。

## 2. 全局模块

系统左侧固定侧边栏包含以下 7 个主模块：

- 员工信息
- 学生管理
- 课程管理
- 薪酬管理
- 数据表
- SOP
- 关于

## 3. 角色与权限

| 角色 | 访问范围 | 特殊权限 |
| --- | --- | --- |
| 超级管理员 | 全部模块 | 设置管理员、设置超级管理员、重置全部账号密码、注销账号 |
| 管理员 | 全部业务模块 | 设置管理员 |
| 一般成员 | 员工信息、学生管理、课程管理、数据表、SOP | 不可访问薪酬管理；课程大纲只读 |
| 访客 | SOP、关于 | 无内部数据访问权，无日志访问权 |

## 4. 全局业务规则

### 4.1 鉴权

- 除 `SOP` 与 `关于` 外，其余页面全部需要登录后访问
- 页面级访问和按钮级操作都要做权限判断
- 未登录用户统一视为访客
- 访客点击受限模块时展示无权限页，并允许跳转登录页

### 4.2 标识符生成

- 员工工号：`YYNNN`
- 学生学号：`YYNNNN`
- 课程编号：`TTKKYYNNN`
- 以上编号都必须满足“删除后不回收序号”

### 4.3 日志

- 保留最近 `180` 天日志
- 新增、编辑、删除、结算、权限变更、注册、注销都要记录
- “关于”页提供日志入口，但先鉴权后跳转

### 4.4 检索

- 普通搜索采用字符相对顺序匹配，不区分大小写
- 高级搜索采用条件组合筛选
- 列表默认每页 `50` 条

## 5. 从原图提炼出的统一视觉规范

来源主要对应原文图 `1、4、5、6、9、12、13、17、20、22、23、24、25、26、27、31`。

### 5.1 页面骨架

- 整体背景使用浅冷灰色
- 左侧为深海军蓝侧边栏，右侧为浅色内容区
- 侧边栏顶部放 Logo 和“研录中台”标题
- 当前模块采用整行蓝色高亮，不只是文字变色
- 侧边栏底部固定用户区域，与主导航区用分割线隔开

### 5.2 内容区

- 页面标题使用大号粗体中文标题，左上对齐
- 内容区主要组件为白底卡片、表格、弹窗
- 大面积内容面板统一使用圆角和轻阴影，避免纯平铺

### 5.3 按钮体系

- 普通按钮：白底、浅灰边框
- 主按钮：亮蓝色实底
- 危险按钮：红色或珊瑚红实底
- 表格工具按钮统一高度，同行左对齐，搜索框与主按钮组分离

### 5.4 表格体系

- 表头加粗
- 除薪酬管理外，第一列统一为复选框
- 单元格留白充足，行高偏舒展，不走紧凑后台风格
- 搜索输入框不贴按钮组，右侧预留明确空隙

### 5.5 弹窗体系

- 弹窗以白色长卡片呈现，圆角较明显
- 录入型弹窗多为双列表单布局
- 内容多时弹窗内部滚动，不要把浏览器窗口直接撑爆
- 底部操作按钮采用右下角固定式排列

### 5.6 卡片入口体系

- 数据表与 SOP 使用三列卡片网格
- 默认白卡片 + 轻阴影
- hover 时卡片轻微上浮、阴影增强
- 数据表 hover 高亮边框为蓝色
- SOP hover 高亮边框为绿色

## 6. 全局交互基线

- 所有删除动作必须二次确认
- 选择日期、时间、年月的场景统一使用可视化日期组件
- 单条勾选允许查看/编辑
- 多条勾选默认禁用查看/编辑，避免批量误操作
- 能自动计算的字段统一只读展示，不允许手填破坏规则

## 7. 全局实施顺序

1. 先做鉴权、布局、通用组件
2. 再做员工和用户管理
3. 再做学生管理
4. 再做课程大纲
5. 再做课程详细信息与选课
6. 再做薪酬管理
7. 再做数据表、SOP、关于
8. 最后做移动端统一适配

## 8. 文档使用约束

- 各阶段文档默认继承本文件的权限、视觉和交互规范
- 后续如有实现细节冲突，以“更具体的阶段文档”为准
