## 阶段路线

1. `Phase 0`：先搭登录、权限、路由、布局和通用组件
2. `Phase 1`：落员工信息与后台账号/权限管理
3. `Phase 2`：落学生管理与服务档案
4. `Phase 3`：落课程大纲版本化管理
5. `Phase 4`：落课程详细信息、学生选课与高级检索
6. `Phase 5`：落薪酬汇总与结算
7. `Phase 6`：落数据表、SOP、关于、日志入口
8. `Phase 7`：做移动端适配与体验打磨

## 设计吸收原则

- 所有阶段文档都已经吸收原始 `docx` 中对应图片的 UI 结构
- 图片不再只作为“参考图”，而是被转成页面布局、交互状态、按钮位置、视觉语言和验收规则
- 同一类页面统一遵循一个设计系统，避免每个模块各做各的
