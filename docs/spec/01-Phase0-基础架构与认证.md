# 01. Phase 0 - 基础架构与认证

## 1. 阶段目标

先建立整个中台的通用骨架，保证后续业务模块都站在统一的登录、权限、布局、组件和视觉基础上开发。

## 2. 本阶段范围

- 登录页
- 访客态/无权限态
- 框架级路由与权限守卫
- 左侧导航与右侧内容布局
- 登录状态保持
- 用户状态区域
- 通用按钮、表格、分页、弹窗、日期组件

## 4. 页面规格

### 4.1 登录页

依据图4，登录页采用“居中白色卡片 + 顶部 Logo + 标题 + 纵向表单”的结构。

- 卡片居中，四周留较大空白
- 顶部展示 `res/logo.png`
- 标题为“欢迎登录研录教学管理中台”
- 输入项顺序：手机号、密码
- “保留登录状态”默认勾选
- 同行右侧预留“忘记密码？”文案入口
- 登录按钮为蓝色主按钮
- 注册按钮为白色普通按钮
- 点击注册时不是进入注册页，而是弹出提示：联系超级管理员添加账号、密码，或发邮件到指定邮箱

### 4.2 主框架布局

依据图1，主框架固定为：

- 左侧深色导航栏
- 中部/右侧主内容区
- 侧边栏底部固定当前用户信息

侧边栏视觉约束：

- 顶部 Logo 居中
- “研录中台”为大号白字
- 主导航项纵向排列
- 激活态为整行亮蓝底，不仅是左边一条激活线
- 用户区域与导航之间有明确分割线

### 4.3 访客态与无权限态

依据图5：

- 未登录时访问业务数据页，不显示空白表格，而显示无权限页
- 内容区中间展示锁图标、标题“无访问权限”和说明文字
- 左下角用户区显示红点 + “访客（点击登录）”
- 点击该区域直接跳转登录页

### 4.4 用户状态气泡

依据图26、图27：

- 已登录时，侧边栏底部用户名左侧显示绿点
- 鼠标悬停用户名，向上弹出白色气泡卡片
- 气泡顶部展示身份文案，如“身份：超级管理员”
- 气泡内展示“用户设置”和“退出登录”
- 未登录时不出现气泡

## 5. 权限与路由规格

### 5.1 路由守卫

- 全局路由守卫在进入页面前执行
- 对 `SOP` 和 `关于` 之外的页面强制检查登录态
- 对薪酬管理、日志入口、课程大纲编辑等页面继续检查角色权限

### 5.2 会话保持

- 勾选“保留登录状态”后允许持久会话
- 刷新页面后若会话有效，应恢复登录状态和当前用户信息
- 若会话失效，直接回到登录页或无权限页

## 6. 通用组件规格

### 6.1 按钮

- `default`：白底、浅灰边框
- `primary`：蓝底白字
- `danger`：红底白字

### 6.2 表格

- 表头加粗
- 默认首列复选框
- 支持禁用态按钮联动
- 支持分页

### 6.3 弹窗

- 白色圆角卡片
- 支持内部滚动
- 底部操作区右对齐

### 6.4 日期类组件

- 日期、时间、月份统一用可视化选择器
- 不允许让用户直接手输自由文本替代日期控件

## 7. 本阶段验收标准

- 未登录访问业务页时能被正确拦截
- 登录页 UI 与图4基本结构一致
- 主框架布局和侧边栏结构与图1一致
- 访客态文案、锁图标、左下角访客入口与图5一致
- 已登录气泡菜单可正确展示身份、用户设置、退出登录
- 后续模块可直接复用按钮、表格、弹窗和日期组件
