# 04. Phase 3 - 课程大纲管理

## 1. 阶段目标

先把课程板块和二级课程类别做标准化，再让课程详细信息依附在稳定的大纲体系下运行。

## 2. 本阶段范围

- 课程大纲页
- 版本切换
- 向大纲添加
- 从大纲删除
- 创建新大纲
- 导入并覆盖
- 下载空白模板
- 删除当前大纲

- 图11：课程管理进入课程大纲的方式
- 图12：课程大纲页面

## 4. 页面结构规格

依据图12：

- 页面标题：`研录课程大纲`
- 顶部工具条包含：
  - 大纲版本下拉框
  - 编辑
  - 向大纲添加
  - 从大纲删除
  - 创建新大纲
  - 导入并覆盖
  - 下载空白大纲模板
- 主体内容不是单一表格，而是按“课程所属板块”分组展开
- 每个板块是独立白色卡片区块
- 区块标题示例：`GPA提升 (GP)`、`科研赋能 (KY)`
- 板块内表格字段为：复选框、序列号、二级课程类别名称、建议授课方式、计划授课老师、实际授课老师（自动同步）、教案排期

## 5. 数据规则

- 大纲版本命名规则：`课程大纲-YYA`
- 同一年内，`A/B/C...` 顺序递增
- 新版本命名依据“当前最新版本”自动推算
- 课程所属板块存储两位字母标识符
- 序列号为两位正整数
- 计划授课老师存员工工号，显示时转成姓名
- 内部还需保存空白模板 `res/kcdg.xlsx`

## 6. 核心交互规则

### 6.1 勾选逻辑

- 勾选 `1` 条时启用编辑、从大纲删除
- 勾选 `2` 条及以上时禁用编辑

### 6.2 创建新大纲

- 必须弹警告
- 在当前页面创建空白新大纲
- 默认版本号自动递增

### 6.3 导入并覆盖

- 必须弹警告
- 只能接收空白模板格式导出的 `xlsx`
- 先做解析校验，再允许覆盖
- 版本号保持不变，只替换当前版本内容

### 6.4 从大纲删除

- 删除前必须明确列出将被删除的二级课程类别名称
- 需要提醒：若现有课程引用了该分类，则课程分类将变为空值

### 6.5 删除当前大纲

- 必须弹高风险警告
- 必须要求输入当前版本号进行确认
- 校验版本号匹配后才允许删除

## 7. UI 细节约束

- 顶部工具条应整齐排列在同一行
- “下载空白大纲模板”为蓝色文本链接风格，不强行做按钮
- 分组卡片上下留白较大，形成明显版块层次
- “实际授课老师（自动同步）”一列不允许因为内容过长把行高撑大，应支持点击查看完整名单

## 8. 本阶段验收标准

- 可创建、切换、编辑多个大纲版本
- 新版本号推算正确
- 导入只接受合法模板并先校验后落库
- 删除大纲条目和删除整版大纲都具备高风险确认
- 大纲页按课程所属板块分组展示，不是扁平无层次表格
