# 08. Phase 7 - 移动端适配

## 1. 阶段目标

保证中台在手机浏览器中具备可用性，尤其保证课程管理相关高频操作可完成。

## 2. 本阶段范围

- 侧边栏移动端改造
- 列表字段裁剪
- 搜索入口移动化
- 弹窗与选择器响应式
- 课程管理关键操作移动端优化

## 3. 对应原图

- 图31：移动端展开侧边栏示例

## 4. 移动端总体布局

依据图31：

- 侧边栏不再始终展开
- 左下角常驻 `≡` 图标
- 点击后从左侧拉出完整侧边栏
- 展开的侧边栏仍保留桌面端的深色视觉和蓝色激活态
- 左下角用户信息区域继续保留在侧边栏底部

## 5. 移动端列表裁剪规则

### 5.1 员工信息

仅展示核心字段：

- 员工姓名
- 具体工作职责
- 电话
- 员工来源

### 5.2 学生管理

仅展示核心字段：

- 学生姓名
- 年级
- 专业
- 公共课剩余
- 1v1剩余

### 5.3 课程管理

仅展示核心字段：

- 课程名称
- 所属板块
- 计划授课时间
- 实际授课老师

### 5.4 其他页面

- 不强制重做为移动卡片流
- 允许横向滚动，但要限制整体宽度和溢出观感

## 6. 控件适配规则

- 桌面端搜索框在移动端收敛为搜索按钮或搜索图标
- 工具按钮保持单行优先，不够时允许横向滚动
- 弹窗改为更高占比的移动端面板，确保表单可滚动
- 日期选择器、学生选择器、员工选择器都必须触屏友好

## 7. 课程管理的特别要求

移动端至少要保证以下动作顺畅：

- 添加课程
- 查看课程
- 修改课程
- 选择授课老师
- 选择选课学生
- 录入授课时长
- 查看自动计算的课时和状态

## 8. 视觉延续规则

- 保留桌面端的深色侧边栏和亮蓝激活色
- 不把移动端做成完全不同的设计语言
- 表格宽度缩短，但不要牺牲主按钮可点击性

## 9. 本阶段验收标准

- 手机浏览器中可打开、关闭侧边栏
- 课程管理的新增、查看、编辑可完整走通
- 搜索入口可在手机上正常触发
- 关键弹窗在手机上不会出现按钮被遮挡或无法提交的问题
