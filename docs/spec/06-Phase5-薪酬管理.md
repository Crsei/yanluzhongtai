# 06. Phase 5 - 薪酬管理

## 1. 阶段目标

基于课程实际授课数据，自动形成老师在指定时间范围内的待结算薪酬记录，并支持手工补录和结算流水留痕。

## 2. 本阶段范围

- 薪酬列表
- 本月/上月/自定义时间范围切换
- 仅看薪资未结清筛选
- 查看课程
- 结算
- 手动添加记录
- 删除手动记录

## 3. 对应原图

- 图17：员工薪酬管理页
- 图18：自定义日期范围选择

## 4. 页面布局规格

依据图17：

- 页面标题：`员工薪酬管理`
- 搜索框位于左侧，支持老师姓名/工号检索
- 其右侧为单选筛选：本月、上月、自定义
- 同行还可叠加“仅查看薪资未结清”
- 右上角放蓝色“手动添加记录”按钮
- 表格首列不是复选框，而是操作列

## 5. 列表字段规格

表格字段至少包含：

- 操作
- 工号
- 老师姓名
- 所属年月
- 单位课时费
- 已授课时
- 总课时费
- 其他劳务
- 其他扣除
- 应结算薪资
- 已结算薪资

补充要求：

- “所属年月”展示格式为 `YYYYMM`
- 应结算薪资使用加粗红字
- 表内金额字段后方需要明确单位“元”

## 6. 操作列规则

- 自动汇总记录：显示“查看课程”和“结算”
- 手动添加记录：把“查看课程”替换为红色“删除记录”

## 7. 结算与查看课程

### 7.1 查看课程

- 点击后弹窗展示当前时间范围内该老师上过的全部课程
- 课程列表至少展示课程名称、计划时间、课时、学生、授课方式

### 7.2 结算

依据图19：

- 弹窗显示老师姓名、应结算总额、此前已结算金额
- 可输入本次结算金额
- 输入框最大值 = 应结算总额 - 此前已结算金额
- 输入框后方显示“元”
- 提交后写入结算记录，并同步记入日志

## 8. 手动添加记录

- 点击“手动添加记录”先弹二次确认
- 文案需明确说明：手动添加的记录无法联动计算课时费，仅是强制添加一条记录
- 手动记录必填字段：
  - 员工
  - 所属年月
  - 其他劳务
  - 其他扣除
- 所属年月使用“精确到月份”的日期组件，存储时转换成 `YYYYMM`
- 其他劳务必须大于 `0`
- 其他扣除不得等于其他劳务

## 9. 数据计算规则

### 9.1 应结算总薪资

`应结算总薪资 = 单位课时费 × 已授课时 + 其他劳务 - 其他扣除`

### 9.2 历史值带出

- 若存在最近一次历史结算记录，则单位课时费默认带出最近值
- 每个月每位老师只允许一个确定的单位课时费

### 9.3 数据来源

- 薪酬页自动汇总当前筛选时间范围内有授课记录的老师
- 已授课时来自课程详情中的课时累计

## 10. 排序规则

- 列表按老师姓名拼音升序

## 11. 本阶段验收标准

- 自动汇总记录可按月或自定义时间正确生成
- 结算弹窗上限校验正确
- 手动记录和自动记录在操作列上可区分
- 所属年月展示为 `YYYYMM`
- 结算行为和手动记录变更全部写入日志
